1- Crie a pasta => mkdir -p ~/dns/bind
	-O parâmetro -p significa que, se algum diretório no caminho já existir, ele não dará erro, e se algum diretório intermediário do caminho não
	existir, ele será criado automaticamente.
	
2- Entre na pasta => cd ~/dns/bind

3- Crie o arquivo => vi named.conf

options {
directory "/var/cache/bind";
// Para permitir consultas de qualquer lugar (ajuste conforme necessário)
allow-query { any; };
 // Outras opções de configuração podem ser adicionadas aqui
};
zone "exemplo.com" {
    type master;
    file "/etc/bind/db.exemplo.com";
};

4- Crie a pasta "zones" dentro da pasta dns/bind

5- Crie o arquivo "db.exemplo.com" dentro da pasta dns/bind/zones

TTL    604800
@       IN      SOA     ns.exemplo.com. admin.exemplo.com. (
                              2         ; Serial
                         604800         ; Refresh
                          86400         ; Retry
                        2419200         ; Expire
                         604800 )       ; Negative Cache TTL
;

; Nome do servidor
@       IN      NS      ns.exemplo.com.
ns      IN      A       127.0.0.1
@       IN      A       127.0.0.1

6- Crie o arquivo "Dockerfile", dentro da pasta dns/bind 

FROM ubuntu:20.04
RUN apt-get update && \
    apt-get install -y bind9 && \
    apt-get clean
COPY named.conf /etc/bind/
COPY db.exemplo.com /etc/bind/db.exemplo.com
CMD ["named", "-g", "-u", "bind"]

7- docker build -t meu-dns-server . => Construi e a imagem Docker 

8- docker run -d --name dns-server -p 53:53/udp -p 53:53/tcp meu-dns-server => execute o container

	-docker run: Inicia um novo contêiner a partir de uma imagem.
	-d: Executa o contêiner em modo detached (em segundo plano), permitindo que o terminal fique liberado para outros comandos.
	--name dns-server: Dá ao contêiner o nome dns-server. Isso facilita a identificação do contêiner para paradas e remoções futuras.
	-p 53:53/udp e -p 53:53/tcp: Faz o mapeamento das portas 53 (que é a porta padrão do DNS) do host para o contêiner, tanto para conexões UDP
	quanto TCP, necessárias para o funcionamento completo do DNS.
	-meu-dns-server: Nome da imagem que está sendo usada para criar o contêiner. Certifique-se de que my-dns-server é o nome correto da sua
	imagem local.

9- dig @127.0.0.1 exemplo.com ou nslookup exemplo.com 127.0.0.1 => para testar o servidor dns
